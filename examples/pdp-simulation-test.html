PDP Simulation Test Page

Simulates a VTEX Product Detail Page for testing auto-generated conversation starters

How it works

  1. This page must be served at a URL ending in /{slug}/p
  2. It injects three product data sources: window.__NEXT_DATA__, fakes the VTEX IS API, and ld+json
  3. The webchat tries each source in order: __NEXT_DATA__ → IS API → ld+json
  4. To test fallback, remove the __NEXT_DATA__ block → falls to IS API. Remove both → falls to ld+json
  5. SKU resolution is always from ld+json only (independent of product data source)

Setup Steps

  1. Build: npm run build:standalone
  2. Set your CHANNEL_UUID below
  3. Serve this folder with a local server that supports URL rewriting:
  4. npx serve dist-standalone --single or use the script below
  5. Open: http://localhost:3000/ipad-10th-gen/p
